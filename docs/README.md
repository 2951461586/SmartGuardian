> 状态：收口整理版

## 阅读顺序

1. 项目评审：先读 `00-项目总览`、`01-顶层设计`、`04-图册与可视化`。
2. 开发联调：先读 `03-接口文档`、`02-数据库设计`、`08-优化改造`。
3. 测试上线：先读 `05-测试文档`、`06-部署与运维`。
4. 历史追溯：只读 `99-历史版本` 与 `08-优化改造/历史执行记录`。

```text

├─ 02-数据库设计/      数据库设计、字段字典、SQL 脚本
├─ 03-接口文档/        API 清单、OpenAPI、错误码、联调说明
├─ 04-图册与可视化/    Mermaid 图册
├─ 05-测试文档/        测试用例
├─ 06-部署与运维/      部署、上线、巡检
├─ 07-组件资源/        原型图、图标、字体、第三方资源许可
├─ 08-优化改造/        当前架构改造方案、执行记录和门禁说明
└─ 99-历史版本/        已收口、过期或仅保留追溯价值的历史报告
```
