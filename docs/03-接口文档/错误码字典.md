# 学生智慧托管系统错误码字典

> 项目名称：基于鸿蒙系统的学生智慧托管系统  
> 文档类型：错误码字典  
> 版本：V1.2  
> 日期：2026-04-16

---

## 一、通用错误码

| 错误码 | 含义 | 说明 |
|---|---|---|
| 200 | 成功 | 请求处理成功 |
| 400 | 参数错误 | 请求参数不合法 |
| 401 | 未认证 | Token 无效或已过期 |
| 403 | 无权限 | 当前角色不允许执行操作 |
| 404 | 资源不存在 | 查询对象不存在 |
| 409 | 状态冲突 | 业务状态不允许当前操作 |
| 429 | 请求过于频繁 | 超过限流阈值 |
| 500 | 系统异常 | 服务内部错误 |

## 二、业务错误码建议

| 错误码 | 含义 | 适用场景 |
|---|---|---|
| SG1001 | 学生不存在 | 学生档案查询/下单 |
| SG1002 | 监护关系不存在 | 家长查看学生信息 |
| SG2001 | 服务产品已下架 | 下单场景 |
| SG2002 | 班次容量已满 | 排课/预约 |
| SG3001 | 订单未审核通过 | 签到前校验 |
| SG3002 | 订单不可退费 | 退费申请 |
| SG4001 | 非法签到时间窗 | 签到 |
| SG4002 | 重复签到 | 签到 |
| SG4003 | 签到设备未授权 | 签到 |
| SG5001 | 作业任务不存在 | 作业反馈 |
| SG5002 | 家长已确认不可重复提交 | 家长确认 |
| SG6001 | 消息发送失败 | IM 消息 |

## 三、处理建议

- 前端根据错误码进行友好提示与重试控制。  
- 对 `429` 和部分可恢复错误建议启用退避重试。  
- 对安全类错误（401/403）记录审计日志。  
- 对签到与支付场景的错误需保留完整请求链路用于排查。
