# SmartGuardian AGC 运维巡检手册

> 更新日期：2026-04-26  
> 运行主线：AGC Serverless + Cloud Functions + Cloud DB

## 1. 巡检目标

保障智慧学生托管系统在 AGC Serverless 架构下保持稳定、可用、安全和可追踪。

## 2. 每日巡检

- Cloud Functions 调用成功率、错误率、超时率。
- API Gateway 路由可用性和鉴权失败比例。
- Cloud DB 读写成功率、热点集合、慢查询或配额风险。
- 账号、会话、通知端点注册链路是否正常。
- Push/本地通知点击后是否能恢复到目标业务页面。

## 3. 高峰期巡检

- 放学前后重点关注签到、接送码、消息通知、异常预警链路。
- 检查 Cloud Functions 并发、冷启动、错误日志。
- 检查 Cloud DB 关键集合：学生、班次、考勤、消息、订单、退款。
- 检查 AGC API Client 凭证有效期和权限。

## 4. 故障处理

- 登录失败：先检查 AGC 客户端配置、Gateway 地址、Auth/会话云函数日志。
- 业务接口失败：检查对应云函数 domain、路由契约和 Cloud DB 权限。
- 数据缺失：检查 Cloud DB Zone、对象类型版本和种子导入记录。
- 通知异常：检查设备端通知权限、端点注册记录和通知路由参数。
- 构建异常：先运行 `scripts\run-refactor-gates.ps1`，再查看 `build-warning-normal.log`。

## 5. 已退役内容

- `backend/` Spring Boot 服务不再巡检。
- MySQL、Redis、MQ、Jar/Nginx 旧链路不再作为当前系统运维对象。
- 历史数据迁移仅保留 `legacyId` 对账语义，不再依赖旧 Java 服务运行。
