# 学生智慧托管系统运维巡检手册

> 项目名称：基于鸿蒙系统的学生智慧托管系统  
> 文档类型：运维巡检手册  
> 版本：V1.2  
> 日期：2026-04-16

---

## 一、巡检目标

保障智慧托管系统在日常运行与放学高峰期保持稳定、可用、安全、可追踪。

## 二、日常巡检项

### 2.1 服务健康
- 检查网关、认证、订单、考勤、消息、报表等服务进程状态
- 检查接口错误率、超时率、响应时间

### 2.2 数据库
- 检查主从状态、连接数、慢 SQL、磁盘空间
- 检查考勤、消息、时间线等大表增长情况

### 2.3 缓存与消息队列
- 检查 Redis 内存使用率、热点 Key、命中率
- 检查 MQ 堆积、消费失败、重试次数

### 2.4 安全与日志
- 检查异常登录、越权访问、接口安全告警
- 检查审计日志与错误日志是否完整

## 三、高峰期专项巡检

- 重点关注签到接口 TPS 与成功率
- 检查预警通知下发延迟
- 检查 Redis 热点访问与数据库写入压力
- 检查 MQ 是否出现积压

## 四、故障处理建议

- 接口延迟升高：优先检查缓存、数据库慢查询与 MQ 堵塞
- 签到失败率升高：检查时间窗校验、设备鉴权、班次数据缓存
- 消息通知异常：检查消息服务、推送通道与消费队列
- 报表延迟：检查异步统计任务与聚合作业执行状态

## 五、巡检频率建议

- 日常：每日一次
- 高峰期：放学前后重点巡检
- 大版本上线：上线前后加强巡检与回归验证
