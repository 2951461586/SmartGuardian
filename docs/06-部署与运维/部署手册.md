# SmartGuardian AGC Serverless 部署手册

> 适用范围：HarmonyOS 前端 + AGC Cloud Functions + AGC Cloud DB  
> 更新日期：2026-04-26  
> 状态：`backend/` Spring Boot 后端已退役，不再作为主线部署组件。

## 1. 主线架构

- 前端：HarmonyOS Stage 应用，默认 `AGC_SERVERLESS`，通过 AGC Cloud Functions SDK 调用云函数。
- 后端：`cloud-functions/` 下的 AGC Cloud Functions 分域函数。
- 数据：AGC Cloud DB，当前开发 Zone 为 `ClouddbDev`。
- 配置：前端运行时优先配置 AGC 云函数触发器标识；只有部署了 API Gateway 时才需要额外填写网关地址，不再填写 `http://127.0.0.1:8080`。

## 2. 前置条件

- DevEco Studio 与 HarmonyOS SDK 可用。
- AGC 项目已创建应用并下载真实 `agconnect-services.json`。
- AGC Cloud DB 已创建 Zone，并导入对象类型。
- AGC Cloud Functions 已部署或本地 smoke 可通过。
- 本地环境变量已配置 AGC API Client 凭证和 Cloud DB Zone。

## 3. Cloud DB 初始化

开发环境可使用种子脚本重置数据：

```powershell
node scripts\apply-cloud-db-seed.js --reset
```

脚本按集合依赖顺序导入数据，执行前确认：

- `AGC_CLOUD_DB_ZONE=ClouddbDev`
- AGC API Client 凭证文件存在且可访问
- Cloud DB 对象类型与本地 `cloud-functions/cloud-db/schema.json` 对齐

## 4. 云函数部署与验证

云函数源码位于 `cloud-functions/`，分域函数清单由 `cloud-functions/function-manifest.json` 管理。

本地一致性与 smoke 验证：

```powershell
powershell -ExecutionPolicy Bypass -File scripts\check-agc-consistency.ps1
node scripts\cloud-functions-smoke.js
```

部署到 AGC 后，确认每个函数的触发器标识与前端默认值一致，例如 `smartguardian-auth-$latest`；只有额外启用 API Gateway 时才需要配置网关地址。

## 5. 前端构建

```powershell
.\hvigorw.bat assembleHap
```

完整治理门禁：

```powershell
powershell -ExecutionPolicy Bypass -File scripts\run-refactor-gates.ps1
```

## 6. 签名配置

当前仓库不提交真实签名口令和敏感材料。需要通过 DevEco Studio：

1. 打开 `Project Structure > Signing Configs`。
2. 导入或重新生成 HarmonyOS 签名材料。
3. 让 DevEco 写入匹配当前材料的加密 `storePassword` 与 `keyPassword`。
4. 再执行 `hvigorw.bat assembleHap` 验证。
