# 学生智慧托管系统本地部署手册

> 文档类型：本地部署手册  
> 适用范围：HarmonyOS 前端 + Spring Boot 后端 + MySQL 本地联调  
> 更新日期：2026-04-23

## 1. 目标

当前项目不再维护远端服务器部署链路，统一改为本地构建和本地联调：

- 前端：HarmonyOS Stage 应用，默认连接 `http://127.0.0.1:8080`
- 后端：`backend/` 下的 Spring Boot 3 项目
- 数据库：MySQL 8.x

## 2. 本地环境要求

- JDK 17
- Maven 3.9+
- MySQL 8.x
- DevEco Studio

仓库内已经提供：

- 后端本地启动脚本：`backend/run-local.ps1`
- 后端本地打包脚本：`backend/build-local.ps1`
- MySQL 本地容器示例：`backend/docker-compose.mysql.yml`

## 3. 启动 MySQL

如果本机已经有 MySQL，可直接创建数据库并配置账号。

如果希望快速起一个本地 MySQL，可在 `backend/` 目录执行：

```powershell
docker compose -f docker-compose.mysql.yml up -d
```

默认数据库信息：

- 数据库：`smartguardian`
- 用户名：`smartguardian`
- 密码：`smartguardian123`
- 端口：`3306`

## 4. 启动 Spring Boot 后端

后端默认读取以下本地配置：

- `DB_URL=jdbc:mysql://127.0.0.1:3306/smartguardian?...`
- `DB_USERNAME=smartguardian`
- `DB_PASSWORD=smartguardian123`
- `JWT_SECRET=smartguardian-local-jwt-secret-2026-safe`

直接启动：

```powershell
cd backend
.\run-local.ps1
```

首次启动时会自动执行：

- `schema.sql` 建表
- `DataInitializer` 初始化演示账号和基础业务数据

服务启动后监听：

```text
http://127.0.0.1:8080
```

## 5. 本地打包后端

```powershell
cd backend
.\build-local.ps1
```

如果需要手动指定 JDK 17：

```powershell
.\build-local.ps1 -JavaHome "D:\software\java17"
```

## 6. 前端联调

前端默认真实环境地址已经切到：

```text
http://127.0.0.1:8080
```

登录页里的“联调设置”面板可以继续切换：

- `模拟演示`
- `本地后端`

推荐联调顺序：

1. 启动 MySQL
2. 启动 Spring Boot 后端
3. 在 DevEco Studio 构建并运行 HAP
4. 登录页确认真实环境地址为 `http://127.0.0.1:8080`
5. 再进入登录、首页、订单、考勤、作业、消息等流程验证

## 8. 说明

- 原远端 Nginx / Jar 部署链路已停用，不再作为默认运行方式。
- 若后续需要发布环境，请在新的发布方案中单独设计，不要复用旧的远端目录与脚本。
