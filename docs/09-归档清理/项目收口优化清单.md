### 1.2 API路由统一化修复

#### 问题：退费服务路由不符合OpenAPI规范

**当前实现**（错误）：
```typescript
// entry/src/main/ets/services/api/refund.ts
export class RefundService {
  private static BASE_URL = '/api/refunds';  // ❌ 缺少/api/v1
  
  static async createRefund(params: CreateRefundParams): Promise<ApiResponse<RefundRecord>> {
    return post<RefundRecord>(this.BASE_URL, params);
  }
}
```

**OpenAPI规范**（正确）：
```yaml
# docs/03-接口文档/openapi.yaml
/api/v1/orders/{orderId}/refund:
  post:
    tags: [Orders]
    summary: 订单退费
```

**修复方案**：
1. 删除独立的 `RefundService`
2. 在 `OrderService` 中使用已有的 `refundOrder` 方法
3. 如需保留独立的退费管理页面，在 `Entry/src/main/ets/services/api/refund.ts` 中改为调用 `OrderService.refundOrder`

### 1.3 卡片API端点统一

**当前实现**（与OpenAPI不符）：
```typescript
// entry/src/main/ets/services/api/card.ts
export class CardService {
  static async getAbnormalAlerts(): Promise<ApiResponse<AbnormalAlertCard[]>> {
    return get<AbnormalAlertCard[]>('/api/v1/cards/abnormal-alerts', params);  // ❌ 复数
  }
  
  static async getAllCards(): Promise<ApiResponse<TodayStatusCard[]>> {
    return get<TodayStatusCard[]>('/api/v1/cards/all');  // ❌ OpenAPI未定义
  }
  
  static async updateCard(data: CardUpdateRequest): Promise<ApiResponse<CardDataResponse>> {
    return post<CardDataResponse>('/api/v1/cards/update', data);  // ❌ OpenAPI未定义
  }
}
```

**OpenAPI规范**：
```yaml
/api/v1/cards/today-status:
  get:
    tags: [Cards]
    summary: 获取今日托管状态卡片数据
    
/api/v1/cards/abnormal-alert:  # 单数形式
  get:
    tags: [Cards]
    summary: 异常提醒卡片摘要
```

**修复方案**：
1. 将 `getAbnormalAlerts` 改为 `getAbnormalAlert`，路由改为 `/api/v1/cards/abnormal-alert`
2. 删除 `getAllCards` 和 `updateCard` 方法（未在OpenAPI中定义）
3. 如需保留这些功能，需先更新OpenAPI文档

## 九、总结与建议

### 9.1 本次优化价值

✅ **消除混乱**：解决API服务重复定义问题，避免维护困惑  
✅ **规范统一**：统一API路由规范，提升代码一致性

### 9.2 后续维护建议

1. **API开发规范**
   - 新增API服务前，检查是否已存在同类服务
   - 严格遵循 `/api/v1/` 路由规范
   - 所有新增API必须先更新OpenAPI文档

2. **Code Review 检查项**
   - API路由是否符合规范
   - 是否有重复的服务定义
   - 是否正确使用模型字段（OpenAPI标准字段）

3. **技术债务管理**
   - Phase 5 完成后，与后端对齐退费API规范
   - 考虑将扩展API（如 getAllCards）补充到OpenAPI文档
