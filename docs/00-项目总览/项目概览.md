# 学生智慧托管系统项目概览

> 项目名称：基于鸿蒙系统的学生智慧托管系统  
> 技术栈：HarmonyOS / DevEco Studio / ArkTS / ArkUI / AGC Serverless / Cloud Functions / Cloud DB  
> 版本：V1.4  
> 日期：2026-04-27

---

## 1. 项目背景

为响应“双减”与“5+2”政策要求，建设面向学校、家长、托管机构三方协同的学生智慧托管系统，实现课后服务数字化、透明化、可监管化。

## 2. 项目目标

- 满足学生托管预约、审核、排课、签到、作业辅导、消息互动、数据统计等核心业务。
- 支撑家长、学校、机构、教师、管理员等多角色协同。
- 依托 HarmonyOS 原生能力提升家长端的即时触达和多设备体验。
- 构建基于 AGC Serverless、Cloud Functions 与 Cloud DB 的安全合规托管平台。

## 3. 核心模块

1. 基础管理：多角色登录、权限控制、学生档案管理。  
2. 托管服务：服务内容管理、订单创建/审核/退费、智能排课。  
3. 家校互动：实时消息 IM、作业跟踪审批流、学生动态时间线。  
4. 数据统计：考勤报表、财务收支看板、教师绩效统计。  
5. 安全体系：敏感数据脱敏、传输加密、存储加密、审计追踪。  
6. **告警服务**：异常考勤预警、系统告警管理、告警确认与解决。  
7. **退款服务**：退款申请、退款审核、退款金额计算、退款统计。

## 4. 业务闭环

家长在家长端浏览托管服务并提交预约；机构/学校审核订单并匹配班次；学生到校后完成签到并触发状态更新；教师开展看护、作业辅导与互动反馈；系统沉淀考勤、订单、反馈、消息与时间线数据；最终通过报表与看板为学校、机构和家长提供决策支持。

## 5. 关键指标

- 用户满意度：≥ 85%  
- 核心接口响应时间：≤ 2 秒  
- 页面加载时间：≤ 3 秒  
- 高峰期签到并发：通过云函数分域、接口幂等、分页与异步通知降低集中访问压力  
- 数据安全：未成年人敏感信息最小化采集、分级分域保护
