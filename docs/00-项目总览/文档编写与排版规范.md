## 1. 目的

本文用于规范本项目交付文档的结构、编号、样式与导出形式，确保文档在课程设计、毕业答辩、项目评审和实施交付等场景中具备统一性、专业性和可审阅性。

## 2. 统一封面与文档控制信息

每份正式文档建议在标题下方统一包含以下信息：

- 项目名称
- 文档类型
- 文档编号
- 版本
- 状态（草稿版 / 评审版 / 正式交付版）
- 日期
- 页眉建议
- 页脚建议

推荐格式：

```text
> 项目名称：基于鸿蒙系统的学生智慧托管系统
> 文档类型：XXX
> 文档编号：SG-XXX-001
> 版本：V1.2
> 状态：正式交付版
> 日期：2026-04-16
> 页眉建议：基于鸿蒙系统的学生智慧托管系统｜XXX
> 页脚建议：SmartGuardian 正式交付版｜第 X 页 / 共 Y 页
```

## 3. 标题层级与编号规范

- 一级标题使用 `#`
- 二级标题统一使用阿拉伯数字，如 `## 1. 项目背景`
- 三级标题使用分级编号，如 `### 1.1 建设背景`
- 四级标题使用 `#### 1.1.1` 或“场景一/场景二”等形式

正文列表建议：
- 描述性列表采用无序列表
- 流程性内容采用有序列表
- 核心结论建议加粗

## 4. 页眉页脚规范

由于 Markdown 原生不支持固定页眉页脚，建议在导出为 Word/PDF 时按以下格式设置：

### 4.1 页眉建议
- 左侧：基于鸿蒙系统的学生智慧托管系统
- 右侧：具体文档名称

示例：
- `基于鸿蒙系统的学生智慧托管系统｜顶层设计方案`
- `基于鸿蒙系统的学生智慧托管系统｜数据库设计说明书`

### 4.2 页脚建议
- 左侧：SmartGuardian 正式交付版
- 中间：版本号 `V1.2`
- 右侧：`第 X 页 / 共 Y 页`

## 5. 字体与段落建议

导出 Word/PDF 时建议：
- 正文：宋体，小四或 12pt
- 一级标题：黑体，16pt
- 二级标题：黑体，14pt
- 三级标题：黑体，12pt
- 行距：1.25 倍或固定值 22 磅
- 页边距：上下 2.54cm，左右 3.17cm

## 6. 表格与图片规范

- 表格标题置于表格上方，格式：`表 3-1 XXX`
- 图标题置于图片下方，格式：`图 4-2 XXX`
- Mermaid 图在 Markdown 保留源码，答辩版可导出为 PNG/SVG 插图
- 表格字段命名建议与 SQL/OpenAPI 保持一致

## 7. Markdown 导出 Word/PDF 建议

推荐导出顺序：
1. 使用 Typora / VS Code Markdown PDF / Pandoc 导出 PDF
2. 导出前检查目录锚点、分页和代码块换页
3. 对答辩版文档建议加上封面、目录页、页码和修订记录

## 8. 文件命名规范

- 中文说明文档采用语义化命名，如：`数据库设计说明书.md`
- 接口原始标准文件保留标准命名：`openapi.yaml`
- 历史版本文档建议以日期或版本后缀区分
