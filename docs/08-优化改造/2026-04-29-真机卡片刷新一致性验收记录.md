# 2026-04-29 真机卡片刷新一致性验收记录

## 验收目标

把桌面元服务卡片刷新链路从“本地 fallback 可展示”推进到“端侧刷新、云端缓存、线上一致性报告可闭环”：

1. `form_cards` 云端缓存可按需刷新并返回一致性报告。
2. 端侧启动、回前台、卡片系统刷新时，使用同一份云端卡片数据更新桌面卡片。
3. `TodayGuardianCard` 与 `AbnormalAlertCard` 都能按真实 `formName` 分流绑定数据。
4. 提供可复跑的验收脚本，区分静态/云端检查和真机 `hdc` 检查。

## 真机验收口径

当前执行环境未提供 `hdc`，所以本次只能完成静态、云端、构建侧验收。连接真机后执行：

```powershell
.\scripts\check-form-card-consistency.ps1 -RequireDevice -RunCloudSmoke -RunBuild
```

人工观察项：

1. 安装 HAP 后登录家长账号。
2. 添加 `TodayGuardianCard` 与 `AbnormalAlertCard` 到桌面。
3. 触发签到/异常事件，或调用 `POST /api/v1/cards/refresh`。
4. 将应用切回前台，再回桌面观察卡片数据刷新。
5. 查询 `GET /api/v1/cards/consistency?refresh=true`，确认 `consistent=true`，且桌面卡片展示与 `freshPayload` 一致。

## 当前边界

- 真机桌面卡片是否即时刷新，仍依赖系统卡片宿主调度和真机环境；仓库内已补齐端侧 `updateForm` 链路与云端一致性报告。
- 异常告警卡当前由同一个 `TodayGuardianFormAbility` 提供数据分流；后续如果要独立生命周期，可新增专属 `AbnormalAlertFormAbility` 并拆分 profile。
