# 2026-04-29 AGC 云端编排四件套收口记录

## 目标

在已有 AGC Cloud Functions、Service SDK、Cloud DB 主线基础上，把横向副作用从业务函数里拆出来，形成可治理的云端编排层：

1. 事件中心 outbox
2. 通知推送云编排
3. 卡片数据云端缓存
4. Cloud Storage 附件元数据能力

## 本批落地

### 1. 事件中心 outbox

- 新增 `cloud-functions/shared/events.js`，提供 `emitDomainEventAsync()`、`emitAndProcessDomainEventAsync()`、`processPendingEventsAsync()`。
- 新增 `cloud-functions/event/` 域：
  - `GET /api/v1/events/outbox`
  - `POST /api/v1/events/process`
- 新增 Cloud DB 集合 `domain_events`。
- 签到、签退、请假、作业创建/状态/反馈/确认、退款创建/取消已投递领域事件。
- 事件消费统一派生：
  - 站内消息
  - 学生时间线
  - 卡片缓存
  - 异常签到告警
  - 通知任务与送达回执

### 2. 通知推送云编排

- 新增 `cloud-functions/notification/` 域：
  - `GET /api/v1/notifications/jobs`
  - `GET /api/v1/notifications/receipts`
  - `GET /api/v1/notifications/preferences`
  - `PUT /api/v1/notifications/preferences`
  - `POST /api/v1/notifications/jobs/{jobId}/retry`
- 新增 Cloud DB 集合：
  - `notification_jobs`
  - `notification_delivery_receipts`
  - `user_notification_preferences`
- 事件消费时根据 `user_sessions` 中的设备端点生成通知任务，当前支持 `PENDING`、`SENT`、`FAILED`、`SUPPRESSED`、`NO_ENDPOINT` 状态。
- 用户免打扰通过 `enabled`、`quietStartTime`、`quietEndTime`、`channels` 控制。

### 3. 卡片数据云端化

- `form_cards` 由事件消费刷新为云端缓存。
- `card` 域查询 `today-status` 时优先读取 `form_cards`，无缓存时即时生成并回填。
- 异常卡片也走 `form_cards` 缓存，和告警事件联动。

### 4. Cloud Storage 附件能力

- 新增 `cloud-functions/storage/` 域：
  - `POST /api/v1/storage/attachments`
  - `POST /api/v1/storage/attachments/{attachmentId}/complete`
  - `GET /api/v1/storage/attachments`
- 新增 Cloud DB 集合 `cloud_attachments`。
- 当前返回 AGC Cloud Storage 元数据与客户端上传策略，真正二进制上传仍应由端侧使用 AGC Cloud Storage SDK 完成。

### 5. 前端合约

- `ApiEndpoints` 新增 `EVENTS`、`NOTIFICATIONS`、`STORAGE` 路由。
- `AgcFunctionContracts` 新增 `event`、`notification`、`storage` 域。
- 新增 `models/cloud.ts` 与 `services/api/cloud.ts`，统一导出 `EventService`、`NotificationCloudService`、`CloudStorageAttachmentService`。

### 6. 文档与门禁

- `openapi-full.yaml` 新增 Events、Notifications、Storage 标签与路径。
- `function-manifest.json`、各域 `contract.json`、Cloud DB schema/collections/seed 均已补齐。
- `scripts/check-production-readiness.ps1` 新增对云端编排集合与函数域的检查。
- `cloud-functions-smoke.js` 覆盖 outbox、通知任务、通知偏好、卡片缓存、附件注册/完成。
