## 四、迁移优先级

### 🔴 P0 - 立即迁移（高频共享）
1. **ParentOrdersPage → OrderStore**
   - 原因: 订单数据多处使用
   - 收益: 订单状态一致性
   - 工作量: 中等

2. **AdminStudentsPage → StudentStore**
   - 原因: 学生数据高频使用
   - 收益: 减少重复API调用

   - 原因: 管理端订单查看
   - 收益: 代码统一性
   - 工作量: 低

## 五、迁移实施方案

### 5.1 标准迁移模式

**以 ParentOrdersPage 为例**:

#### 优化前
```typescript
@Component
struct ParentOrdersPage {
  @State orders: Order[] = [];
  @State loading: boolean = false;
  
  async aboutToAppear() {
    this.loading = true;
    const resp = await OrderService.getOrders({ pageNum: 1 });
    if (resp.code === 0) {
      this.orders = resp.data;
    }
    this.loading = false;
  }
}
```

#### 优化后
```typescript
@Component
struct ParentOrdersPage {
  @StorageLink('orders') orders: Order[] = [];
  @State loading: boolean = false;
  
  async aboutToAppear() {
    // 优先使用缓存
    const cached = OrderStore.getOrders();
    if (cached.length > 0) {
      this.orders = cached;
    }
    
    // 后台刷新
    this.loading = true;
    const resp = await OrderService.getOrders({ pageNum: 1 });
    if (resp.code === 0) {
      OrderStore.setOrders(resp.data);  // 更新Store
    }
    this.loading = false;
  }
}
```

### 5.2 Store 方法增强

#### OrderStore 增强
```typescript
export class OrderStore {
  private static readonly KEY = 'orders';
  
  // 新增：获取当前订单列表
  static getCurrentOrders(): Order[] {
    return AppStorage.get<Order[]>(this.KEY) ?? [];
  }
  
  // 新增：添加新订单
  static addOrder(order: Order): void {
    const orders = this.getCurrentOrders();
    orders.unshift(order);
    AppStorage.setOrCreate(this.KEY, orders);
  }
  
  // 新增：更新订单状态
  static updateOrderStatus(orderId: number, status: string): void {
    const orders = this.getCurrentOrders();
    const index = orders.findIndex(o => o.id === orderId);
    if (index !== -1) {
      orders[index].orderStatus = status;
      AppStorage.setOrCreate(this.KEY, orders);
    }
  }
}
```

## 七、风险与应对

### 7.1 潜在风险
| 风险 | 影响 | 应对措施 |
|------|------|----------|
| 缓存过期 | 数据不准确 | 添加缓存失效时间 |
| 内存占用 | 性能下降 | 限制缓存大小 |
| 状态不同步 | 数据混乱 | 统一使用 Store API |

### 7.2 回滚方案
