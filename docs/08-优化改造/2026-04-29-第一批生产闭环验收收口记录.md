# 2026-04-29 第一批生产闭环验收收口记录

## 目标

把云端化第一批从“代码链路可跑”推进到“生产环境能明确验收、缺项会失败”的状态，覆盖：

1. Push Kit 真 token 与华为 Push 服务端真实送达。
2. 事件 outbox 从手动/请求后处理升级到 AGC 定时/触发式异步消费。
3. 真机元服务卡片刷新与 `form_cards` 线上数据一致。
4. AGC Cloud Storage 真实 bucket 与端侧 SDK 上传链路。

## 本批落地

### 1. 通知推送生产诊断

- `cloud-functions/shared/push.js` 新增 `getPushDeliveryConfig()`。
- `cloud-functions/notification/index.js` 新增：
  - `GET /api/v1/notifications/delivery-config`
- 返回生产推送状态：
  - `deliveryMode`
  - `huaweiPushConfigured`
  - `realDeliveryReady`
  - `appIdConfigured`
  - `clientIdConfigured`
  - `clientSecretConfigured`

生产验收要求：

- `SMARTGUARDIAN_PUSH_DELIVERY_MODE=real`
- `HUAWEI_PUSH_APP_ID`
- `HUAWEI_PUSH_CLIENT_ID`
- `HUAWEI_PUSH_CLIENT_SECRET`

### 2. 事件触发生产诊断

- `cloud-functions/event/index.js` 新增：
  - `GET /api/v1/events/trigger-config`
- `/api/v1/events/trigger` 已强制支持 `SMARTGUARDIAN_EVENT_CONSUMER_SECRET`。
- `cloud-functions-smoke.js` 已覆盖：
  - 未带 secret 调用 trigger 返回 403。
  - 带 `x-smartguardian-event-secret` 调用 trigger 成功。

生产验收要求：

- 配置 `SMARTGUARDIAN_EVENT_CONSUMER_SECRET`。
- AGC 定时触发器或外部调度器调用 `POST /api/v1/events/trigger`。
- Header 使用 `x-smartguardian-event-secret`。

### 3. Storage bucket 生产诊断

- `cloud-functions/storage/index.js` 新增：
  - `GET /api/v1/storage/config`
- 附件注册时支持默认 bucket：
  - `SMARTGUARDIAN_STORAGE_BUCKET`
  - 或 `AGC_CLOUD_STORAGE_BUCKET`
- `cloud_attachments.storageBucket` 不再只能依赖端侧传入。

生产验收要求：

- 配置 `SMARTGUARDIAN_STORAGE_BUCKET` 或 `AGC_CLOUD_STORAGE_BUCKET`。
- 端侧注入真实 `CloudStorageUploadAdapter`，把 `localFileUri` 上传到 `uploadPolicy.storagePath`。

### 4. 第一批生产闭环门禁

新增：

- `scripts/check-production-closure.ps1`

该脚本串联：

- `check-production-readiness.ps1`
- Push 真实送达配置检查
- 事件 trigger secret 检查
- Storage bucket 检查
- Push token provider / session endpoint registration 代码边界检查
- 华为 Push gateway real mode 代码边界检查
- Storage upload adapter 代码边界检查
- Form card consistency gate

生产环境完整验收命令：

```powershell
.\scripts\check-production-closure.ps1 -RequireDevice -RunCloudSmoke -RunBuild
```
