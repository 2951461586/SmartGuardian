# 2026-04-29 云端化能力推进进度与后续方向

## 总体结论

当前工程已经从“端侧 Mock + 同步业务处理”推进到“AGC Cloud Functions + Cloud DB + 事件 outbox + 通知任务 + 卡片缓存 + 附件 metadata”的云端主线。

P0 主链路已经基本落地，并已补齐第一批生产闭环门禁：Push 真实送达配置、事件触发密钥、Storage bucket、卡片真机一致性都纳入 `check-production-closure.ps1`。剩余风险集中在生产环境变量、AGC 控制台触发器和真机观察。P1/P2 主要进入第二阶段：跨设备可恢复状态、定时预聚合、远程配置、Agent 云端策略、对账、异步导出、多机构策略配置。

## 后续改造批次

### 第一批：生产闭环验收

目标是把 P0 从“代码已落”推进到“控制台/真机可验收”。

- Push Kit 真 token：端侧接官方 Push Kit SDK，写入 `user_sessions.notificationToken`。
- 真实推送：配置 Push 服务端凭据，打开 `SMARTGUARDIAN_PUSH_DELIVERY_MODE=real`。
- 事件异步消费：AGC 定时触发器调用 `/api/v1/events/trigger`，并启用 `SMARTGUARDIAN_EVENT_CONSUMER_SECRET`。
- 卡片真机一致性：安装 HAP、添加两类桌面卡片、触发业务事件、比对 `/api/v1/cards/consistency?refresh=true`。
- Storage 真上传：端侧补 AGC Cloud Storage adapter，把 `localFileUri -> storagePath` 的真实上传跑通。

### 第二批：可恢复云状态

目标是把“端侧实时状态”变成“云端可恢复任务”。

- 新增 `handoff_tasks`、`device_sessions`，支撑跨设备预约、支付确认、教师大屏签到进度。
- 新增 `export_tasks`，支撑异步报表导出、文件生成、下载通知。
- 新增支付对账任务和幂等记录，支撑回调验签、失败重试、对账差异处理。

### 第三批：配置与治理云端化

目标是减少代码发布依赖，让运营和安全策略可配置、可审计。

- 远程配置：Agent 开关、LLM provider/model、告警阈值、灰度页面、模块开关。
- Agent 策略：提示词模板、知识库、角色策略、拒答规则、问答审计报表。
- 审计治理：保留策略、分级告警、导出审批、安全事件处理流。
- 多机构策略：服务规则、价格策略、排课容量、审批流、机构级工作台 manifest。
