# 2026-04-27 AGC 客户端密钥管理说明

## 结论

当前项目需要区分三类 AGC 配置文件，不能混用：

- HarmonyOS 客户端配置：`entry/src/main/resources/rawfile/agconnect-services.json`，随 HAP 打包，用于 ArkTS 客户端 SDK 初始化。
- AGC Cloud Server SDK 凭据：`secrets/agc-apiclient-smartguardian.json`，只给 Node 云函数/Cloud DB 管理脚本使用。
- AGC Connect API 服务端认证私钥：形如 `*private.json`，只给服务端管理脚本换取 Connect API access token 使用，不进入 HAP，不提交仓库。

## 本轮落地

- `.gitignore` 已新增 `**/*private*.json`，避免把 AGC Connect API 私钥误提交。
- `cloud-functions/.env.example` 已新增 `AGC_CONNECT_PRIVATE_KEY_FILE` 和 `AGC_CONNECT_JWT_ALG`。
- `scripts/cloud-env-loader.js` 已支持解析 AGC Connect API 私钥路径。
- `scripts/agc-connect-api-client.js` 已支持校验私钥 JSON，并可按需请求 AGC Connect API access token。
- `scripts/check-agc-prerequisites.ps1` 已增加可选的 AGC Connect API 私钥检查。

## 本地配置

`cloud-functions/.env` 中按需增加：

```env
AGC_CONNECT_PRIVATE_KEY_FILE=C:\Users\29514\Downloads\1175541751777292791936private.json
AGC_CONNECT_JWT_ALG=RS256
```

如果官方文档要求使用 `PS256`，只需要改为：

```env
AGC_CONNECT_JWT_ALG=PS256
```

## 使用命令

只校验私钥文件，不请求 token：

```powershell
node scripts\agc-connect-api-client.js --key C:\Users\29514\Downloads\1175541751777292791936private.json
```

请求 AGC Connect API access token：

```powershell
node scripts\agc-connect-api-client.js --key C:\Users\29514\Downloads\1175541751777292791936private.json --token
```

或使用 `.env` 中的 `AGC_CONNECT_PRIVATE_KEY_FILE`：

```powershell
node scripts\agc-connect-api-client.js
```

## 签名说明

缺少真实设备时，DevEco Studio 无法自动创建调试 Profile。模拟器安装 HAP 可以暂时跳过签名；真机调试和上架发布仍需要在 DevEco Studio / AppGallery Connect 中生成证书、Profile 和 signingConfig。

参考：[AGC Connect API 获取服务端认证](https://developer.huawei.com/consumer/cn/doc/app/agc-help-connect-api-obtain-server-auth-0000002271134661#section17426104123117)
