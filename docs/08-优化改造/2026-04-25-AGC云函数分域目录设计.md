# SmartGuardian AGC 云函数分域目录设计

> 日期：2026-04-25  
> 目标：把现有 `Spring Boot Controller + Service` 边界，重排为 `AGC API Gateway + Cloud Functions` 分域结构

> 退役状态：截至 2026-04-26，`backend/` 旧 Spring Boot 包已删除，Cloud Functions 分域结构是唯一后端主线。

## 已落地

✔ 前端请求层已新增 `AgcRequestAdapter`，并按路由前缀解析函数域。  
✔ 云函数域按现有前后端真实业务边界拆分，不再以 Controller 类名为边界。  
✔ API Gateway 继续保留 `/api/v1/*` 外部路由口径，前端服务层暂不需要大面积改接口名。

## 分域映射

| 路由前缀 | 函数域 | 说明 |
| --- | --- | --- |
| `/api/v1/auth/login` `/logout` `/refresh` | `auth` | 登录、会话刷新、退出 |
| `/api/v1/auth/me` `/api/v1/users/*` | `user` | 当前用户、资料、角色上下文 |
| `/api/v1/students/*` | `student` | 学生档案、监护关系 |
| `/api/v1/service-products/*` | `service` | 托管服务产品 |
| `/api/v1/orders/*` | `order` | 订单创建、审核、退款申请 |
| `/api/v1/sessions/*` | `session` | 班次、排班、学生名单 |
| `/api/v1/attendance/*` | `attendance` | 签到、签退、请假、异常 |
| `/api/v1/homework/*` | `homework` | 作业任务、反馈、确认 |
| `/api/v1/messages/*` | `message` | 消息、未读数、批量已读 |
| `/api/v1/reports/*` | `report` | 报表与经营分析 |
| `/api/v1/refunds/*` | `refund` | 退款主流程 |
| `/api/v1/workbench/*` | `workbench` | 三端工作台 manifest |
| `/api/v1/alerts/*` | `alert` | 告警处理 |
| `/api/v1/timeline/*` | `timeline` | 学生成长时间线 |
| `/api/v1/cards/*` | `card` | 卡片型聚合读模型 |

## 目录规范

✔ 每个函数目录只负责一个业务动作，不再混装“列表 + 详情 + 新增 + 审核”。  
✔ 每个函数域统一包含 `contract`、`handler`、`repository`、`mapper` 四层职责。  
✔ `shared/contracts` 统一放返回体、错误码、分页、登录态、租户上下文字段。  
✔ `shared/cloud-db` 统一放 Cloud DB 模型、索引声明、查询封装。  
✔ 聚合读模型优先落在 `workbench / card / report`，避免前端多跳拼装。  

## API Gateway 规则

✔ 外部路径继续保持 REST 风格：`/api/v1/...`。  
✔ Gateway 负责把 REST 路由转发到对应 Cloud Function。  
✔ 前端统一追加以下头，供网关和函数链路追踪使用：

```text
X-SmartGuardian-Request-Mode: AGC_SERVERLESS
X-SmartGuardian-Function-Domain: <domain>
X-SmartGuardian-Function-Name: <functionName>
X-SmartGuardian-Function-Route: <routePath>
```
