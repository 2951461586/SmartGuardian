## 🎯 优化建议

### 策略一：保留有价值的关联字段 ✅ **推荐**

**理由**:
1. **性能优化**: 后端通过JOIN查询返回关联数据，减少前端API调用次数
2. **用户体验**: 数据立即可用，无需等待额外请求
3. **代码简洁**: 避免前端手动关联查询的复杂度

**实施方案**:
1. 为所有字段添加清晰注释，区分"OpenAPI标准字段"和"UI便利字段"
2. 确保后端API返回这些关联字段
3. 前端代码中优先使用关联字段展示

---

### 策略二：分离标准模型和视图模型

**理由**:
1. 类型更清晰，符合接口设计原则
2. 便于API版本管理和兼容性维护

**实施方案**:
```typescript
// 标准模型 - 对应 OpenAPI
export interface Order {
  id: number;
  studentId: number;
  serviceProductId: number;
  // ...只有标准字段
}

// 视图模型 - 包含关联数据
export interface OrderViewModel extends Order {
  studentName?: string;
  serviceProductName?: string;
  guardianName?: string;
}
```

**缺点**: 增加类型定义复杂度，需要额外的转换逻辑。
