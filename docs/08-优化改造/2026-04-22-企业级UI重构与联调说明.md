# 2026-04-22 企业级 UI 重构与联调说明

## 1. 改造目标

本轮改造围绕“企业级教育 HAP 工作台”这个目标，集中处理了三个问题：

- 首页视觉风格分散，缺少统一的企业级工作台层次
- 前端长期以 Mock 为主，真实后端联调需要手改配置文件
- 管理端首页部分统计带有明显演示数据痕迹，缺少真实接口支撑

## 3. UI 设计调整

### 3.1 统一风格

本次将主工作台统一为浅蓝教育科技风格，核心表现为：

- 顶部欢迎区使用浅色渐变、圆角和轻阴影建立工作台层次
- 指标卡统一为大圆角、浅描边、低阴影，减少旧版碎片化观感
- 快捷操作改为卡片式入口，更贴合企业工作台的操作心智
- 底部导航改为悬浮胶囊式容器，突出当前激活页面

### 3.2 三端首页重点

家长端首页：

- 强化“今日托管 + 作业 + 消息 + 异常提醒”的组合
- 首页直接显示异常提醒条和老师反馈

教师端首页：

- 强化“当前班次 + 签到进度 + 待处理消息”
- 快捷入口更明确地服务扫码签到、考勤和作业处理

管理端首页：

- 强化“学生规模 + 今日到班 + 待处理订单 + 净收入”
- 管理首页不再只做静态展示，而是优先拉取真实接口数据

## 4. 前后端联调方式

### 4.1 后端端口

仓库内 SpringBoot 后端默认配置如下：

```yaml
server:
  port: 8080
```

对应文件：

- `backend/src/main/resources/application.yml`

### 4.2 前端运行时切换

前端仍默认保持 `DEV_MOCK`，但新增了运行时联调配置能力。

现在可直接在登录页操作：

1. 打开登录页
2. 在“联调配置”区域切换到“真实联调”
3. 输入真实后端地址，例如：

```text
http://<服务器IP>:8080
```

4. 点击“保存联调地址”
5. 正常登录进入系统

### 4.3 配置存储

运行时配置通过 `AppStorage` 持久化，键如下：

- `smart_guardian_api_env`
- `smart_guardian_api_base_url`

运行时统一入口如下：

- `ApiConfig.getCurrentEnv()`
- `ApiConfig.getConfiguredBaseUrl()`
- `ApiConfig.applyRuntimeConfig(env, baseUrl?)`

### 5.3 管理端首页

- `MessageService.getUnreadCount()`
- `ReportsService.getFinanceReport()`

## 6. 联调建议顺序

建议按以下顺序进行联调验证：

1. `/api/v1/auth/login`
2. `/api/v1/auth/me`
3. 家长端首页四个接口
4. 教师端首页三个接口
5. 管理端首页五个接口
6. 再扩展验证订单、消息、报表等次级页面

## 7. 当前遗留项

- `OrderService.getOrders()` 目前仍按数组模型消费，后续建议统一收敛为分页模型
- 这次优先改造了登录页、主工作台和三端首页，列表页/详情页视觉风格可在下一轮继续统一
- 若后端环境稳定，可考虑在发版前隐藏登录页中的演示账号提示，仅保留联调配置入口
