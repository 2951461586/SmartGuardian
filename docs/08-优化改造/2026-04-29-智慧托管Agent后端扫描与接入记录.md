# 智慧托管 Agent 后端扫描与接入记录

日期：2026-04-29

## 官方能力确认

- 参考华为 HarmonyOS Agent Framework Kit 指南：https://developer.huawei.com/consumer/cn/doc/harmonyos-guides/harmony-agent-framework-kit-guide
- 本地 DevEco SDK 已提供 `@kit.AgentFrameworkKit`，导出 `AgentController`、`FunctionController`、`FunctionComponent`、`ButtonType`、`FunctionOptions`。
- 当前工程已接入 `AgentController.isAgentSupport(context, agentId)` 能力检测，检测失败不阻塞启动；真正的会话入口需要在华为侧完成 Agent ID 发布后再展示 `FunctionComponent`。

## 后端扫描结论

| 领域 | 当前状态 | 本轮处理 |
| --- | --- | --- |
| AGC 云函数域 | 16 个业务域，统一由 `function-manifest.json`、`contract.json`、OpenAPI、前端合约约束 | 新增 `agent` 域，补齐四处合约 |
| 鉴权与作用域 | `shared/router.js` 默认 `requireUser`；角色由路由 `roles` 控制 | Agent 复用鉴权，并按学生、考勤、作业、班次、订单、请假作用域裁剪 |
| 数据读取 | `shared/store.js` 支持本地 seed 与 AGC Cloud DB 切换 | Agent 使用只读聚合，不新增 Cloud DB 集合 |
| LLM 接入 | 之前无统一 LLM 网关 | 新增 OpenAI-compatible 环境变量接入，未配置时走本地规则回答 |
| 自动化能力 | 工作台 manifest 仅提供模块入口 | Agent 新增总结、汇报、问答、自然语言导航 |

## 新增接口

| 方法 | 路径 | 用途 |
| --- | --- | --- |
| POST | `/api/v1/agent/chat` | 托管问答与多意图入口 |
| POST | `/api/v1/agent/summary` | 生成当前角色作用域内托管总结 |
| POST | `/api/v1/agent/report` | 生成托管汇报 |
| POST | `/api/v1/agent/navigation` | 将自然语言解析为页面导航动作 |

## LLM 环境变量

```env
SMARTGUARDIAN_LLM_PROVIDER=baiduqianfancodingplan
SMARTGUARDIAN_LLM_BASE_URL=https://qianfan.baidubce.com/v2/coding
SMARTGUARDIAN_LLM_MODEL=qianfan-code-latest
SMARTGUARDIAN_LLM_API_KEY=<llm-api-key>
SMARTGUARDIAN_LLM_TIMEOUT_MS=30000
SMARTGUARDIAN_LLM_DISABLED=false
```

说明：本地烟测不会读取真实密钥；未配置 `SMARTGUARDIAN_LLM_API_KEY` 时返回 `source=LOCAL_RULES`，配置成功且调用成功时返回 `source=LLM`。
