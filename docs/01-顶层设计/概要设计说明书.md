## 1. 设计目标

在需求规格基础上，对系统模块划分、技术架构、关键流程与设计边界进行概括说明，作为详细设计、开发实现与项目答辩的中间层文档。

## 2. 系统架构概览

系统总体采用“鸿蒙端 + AGC API 网关 + Cloud Functions 分域函数 + Cloud DB 数据层 + 官方 Kit 能力”的分层结构：

- 端侧：家长端、教师端、管理端、服务卡片
- 网关层：AGC API Gateway、统一鉴权、路由与日志
- 服务层：Cloud Functions 分域函数，覆盖认证、学生、订单、排课、考勤、作业、消息、报表
- 数据层：AGC Cloud DB，历史 SQL 仅作为逻辑模型和迁移对照
- 能力层：AGConnect Auth、NotificationKit、Location、会话设备端点注册

## 3. 模块设计

### 3.1 基础管理模块
负责用户、角色、权限、学生档案和监护关系维护。

### 3.2 托管服务模块
负责服务产品配置、订单管理、支付、审核、退费与排课。

### 3.3 家校互动模块
负责 IM 消息、作业反馈、时间线与异常通知。

### 3.4 统计分析模块
负责考勤、财务、绩效和预警数据分析。

### 3.5 告警服务模块
负责系统告警、异常考勤预警的生成、确认、解决与统计。支持活跃告警监控、告警状态流转（活跃→确认→解决/忽略）。

### 3.6 退款服务模块
负责退款申请、审核、金额计算与统计。支持退款预计算、退款记录查询、退款取消与退款数据统计。

## 4. 关键流程设计

### 4.1 订单流程
家长下单 → 支付 → 审核 → 排课 → 执行 → 完成/退费。

### 4.2 签到流程
班次生成 → 学生签到 → 状态更新 → 消息通知 → 数据沉淀 → 异常处理。

### 4.3 作业反馈流程
任务下发 → 教师辅导 → 反馈提交 → 家长确认 → 统计归档。

## 5. 技术选型

- 开发环境：DevEco Studio
- 开发语言：ArkTS
- UI 框架：ArkUI
- 接口形式：RESTful API / JSON
- 后端：AGC Serverless / Cloud Functions
- 数据：AGC Cloud DB
- 契约：OpenAPI 3.0.3 / AGC 函数 manifest

## 6. 性能设计

- 端侧列表懒加载、组件复用
- 云函数分域处理，避免单点巨型服务
- 通知、时间线、统计等非阻塞任务异步化
- 签到接口限流、幂等、防重复提交

## 7. 安全设计

- 统一认证授权
- HTTPS 传输加密
- 敏感字段列级加密与脱敏展示
- 审计日志与权限分级控制

## 8. 可扩展性设计

后续支持元服务卡片增强、分布式流转、AI 智能排课、成长画像分析、更多学校/机构接入等能力。
